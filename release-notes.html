v2.7.9 detailed changelog

What shipped, what improved, and how to upgrade without surprises.

Features
December 2025
  • Manual bounty tracking now stores amounts for better bookkeeping.
  • Attack Tracker keeps a running list of targets you already hit during a chain.
  • Help Center refreshed with clearer shortcuts, rate limit tips, and setup notes.
  • Rate limit popover shows remaining calls and reset timers in real time.
Improvements
Reliability
  • Backup and restore bundles target groups, history, and preferences in one flow.
  • Installer verifies integrity before first launch to catch corrupted downloads.
  • Status bar now calls out invalid API keys and throttling with clearer labels.
  • UI polish for spacing, typography, and screenshot load speed.
Fixes
Stability
  • Refresh pacing tuned to reduce 429 responses during heavy traffic.
  • Attack Tracker updates no longer stall after long sessions.
  • Help Center links now open to the correct panels.
  • Smoother screenshot shimmer for slower network connections.
Upgrade steps
Rollout
  1. Download the v2.7.9 installer from GitHub releases.
  2. Compare the posted hash before running the .exe.
  3. Export a backup so target groups and history are protected.
  4. Install over your current build; re-enter the API key if prompted.
  5. Re-open the rate limit popover to confirm healthy API budget.

Compatibility

Environment and verification

Avoid surprises by confirming OS support, key scopes, and download integrity before you roll out a new build.

Supported setup

  • Windows 10/11 64-bit with a stable local drive (avoid network shares).
  • Torn API key with user, faction, and attacks scopes only.
  • Clock sync enabled so cooldown and travel timers stay accurate.

Verify every download

  • Match the installer hash posted on GitHub releases before running it.
  • Keep installers and backups in separate folders so restores stay clean.
  • Allow list the installer in your AV only after the hash matches.

API health checks

  • Watch the rate limit popover before long sessions to see available calls.
  • Pause auto-refresh during Torn maintenance to protect your key.
  • Re-enter the key when the status bar shows invalid or throttled states.
Known behaviors
  • 429s can still appear if multiple clients share one API key; stagger refresh timers.
  • Some antivirus tools sandbox new executables; allow list after you verify the hash.
