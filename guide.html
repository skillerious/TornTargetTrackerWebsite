Setup

Quickstart checklist

Follow these steps once and you are ready for every chain. Each step matches the app layout so you do not have to hunt for settings.

Step 1

Install and connect

Run the v2.7.9 installer, open the app, and drop in your Torn API key.

  • Use a key with minimal scopes: user, faction, and attacks.
  • The key is encrypted with AES-256-GCM before storage.
Step 2

Build your target set

Import a CSV or paste target IDs, then group them for quick filtering.

  • Color code groups for chains, traveling, or bounty runs.
  • Use quick filters for attackable, hospital, travel, or jail.
Step 3

Calibrate refresh

Set your polling rate to balance freshness with API limits.

  • Use 15s during chains; 45s or higher for idle tracking.
  • Rate limit popover shows remaining calls before a burst.
Workflows

Operate with a repeatable rhythm

Use these patterns during a chain or while prepping targets. They pair with keyboard shortcuts and the VS Code inspired sidebar.

Chain readiness

Stay on top of targets that are attackable right now.

  • Filter to attackable, then sort by level or travel timer.
  • Pin the attackable list so it never scrolls off screen.
  • Open the target profile in one click when you are ready to swing.

Attack review

Keep a clean log so you never double-tap the same target mid-chain.

  • Enable Attack Tracker to mark every completed hit.
  • Use manual bounty tracking to log payouts as you go.
  • Sort attack history by recency to spot repeats.

Backup and restore

Protect your work before long sessions or installs on another machine.

  • Use File > Backup and Restore to export everything.
  • Backups include target groups, history, and preferences.
  • Store the backup offline or in a private cloud folder.
Reliability

Safety and troubleshooting

A few simple habits keep your API key secure and the app responsive.

Stay safe
  • AES-256-GCM encryption and sandboxed renderer keep your key guarded.
  • Run the app from a local drive; avoid shared or synced temp folders.
  • Use the built-in rate limiter. Manual API bursts can lead to 429 errors.
If something feels off
  • Check the status bar for rate limit warnings or API errors.
  • Refresh tokens by re-entering the API key if status reads invalid.
  • Open a GitHub issue with logs attached for repeatable bugs.
Support

File a ticket

Open an issue on GitHub with screenshots and logs.

Report an issue
