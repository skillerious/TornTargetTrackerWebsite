Setup

Quickstart checklist

Follow these steps once and you are ready for every chain. Each step matches the app layout so you do not have to hunt for settings.

Step 1

Install and connect

Run the v2.7.9 installer, open the app, and drop in your Torn API key.

  • Use a key with minimal scopes: user, faction, and attacks.
  • The key is encrypted with AES-256-GCM before storage.
  • Verify the installer hash on GitHub before first launch.
Step 2

Build your target set

Import a CSV or paste target IDs, then group them for quick filtering.

  • Color code groups for chains, traveling, or bounty runs.
  • Use quick filters for attackable, hospital, travel, or jail.
  • Pin your main chain list so it stays visible while you scroll.
Step 3

Calibrate refresh

Set your polling rate to balance freshness with API limits.

  • Use 15s during chains; 45s or higher for idle tracking.
  • Rate limit popover shows remaining calls before a burst.
  • Pause auto-refresh if Torn's API is under maintenance.
Features

Built for chain discipline

Focus on the tools that keep your target pool clean and your API usage under control.

Target intelligence

Keep the right people in view while you rotate through chains.

  • Quick filters spotlight attackable, hospitalized, jailed, or traveling targets.
  • Group priorities and color coding make it obvious who matters most.
  • Inline status chips show timers so you do not need to click profiles.

Attack Tracker + bounties

Log every swing and payout so you never double-tap a target mid-chain.

  • Attack Tracker marks completed hits as you go.
  • Manual bounty tracking records target IDs and amounts for audits.
  • Sort recent attacks to see who is safe to re-engage.

Rate-limit guardrails

Ship fast without burning your key.

  • Popover shows remaining calls and reset timers in real time.
  • Adaptive refresh slows down when the API starts returning 429s.
  • Clear status bar messaging when a key is invalid or throttled.

Backups and workspace

Move machines or recover quickly with a single file.

  • Backups include target groups, history, filters, and preferences.
  • Restore in-place without wiping data you still need.
  • Store backups offline or in a private cloud for safekeeping.
Workflows

Operate with a repeatable rhythm

Use these patterns during a chain or while prepping targets. They pair with keyboard shortcuts and the VS Code inspired sidebar.

Chain readiness

Stay on top of targets that are attackable right now.

  • Filter to attackable, then sort by level or travel timer.
  • Pin the attackable list so it never scrolls off screen.
  • Open the target profile in one click when you are ready to swing.

Attack review

Keep a clean log so you never double-tap the same target mid-chain.

  • Enable Attack Tracker to mark every completed hit.
  • Use manual bounty tracking to log payouts as you go.
  • Sort attack history by recency to spot repeats.

Backup and restore

Protect your work before long sessions or installs on another machine.

  • Use File > Backup and Restore to export everything.
  • Backups include target groups, history, and preferences.
  • Store the backup offline or in a private cloud folder.
Playbook

Chain routine you can reuse

A short loop to prep, run, and close out chains without burning calls or losing track of payouts.

Before the chain

Health check and prep

  • Verify the installer hash and take a quick backup of targets.
  • Confirm the Torn API status and set refresh to a safe cadence.
  • Stage attackable and bounty groups with pinned filters.
During the chain

Stay on the right targets

  • Keep Attack Tracker open so completed hits are instantly marked.
  • Sort by travel or hospital timer to decide who rotates in next.
  • Use manual bounty tracking to log payouts while you move.
After the chain

Reset and recover

  • Switch to a slower refresh interval to protect your API budget.
  • Restore a backup if you reset filters or switch machines.
  • File issues with logs attached for any odd behavior.
Reliability

Safety and troubleshooting

A few simple habits keep your API key secure and the app responsive.

Stay safe
  • AES-256-GCM encryption and sandboxed renderer keep your key guarded.
  • Run the app from a local drive; avoid shared or synced temp folders.
  • Use the built-in rate limiter. Manual API bursts can lead to 429 errors.
  • Match the installer hash and keep backups off the machine you raid on.
If something feels off
  • Check the status bar for rate limit warnings or API errors.
  • Refresh tokens by re-entering the API key if status reads invalid.
  • Open a GitHub issue with logs attached for repeatable bugs.
  • Restart with auto-refresh paused if you see multiple 429s in a row.
Self-check
  • Confirm time sync on Windows so timers and cooldowns stay accurate.
  • Use the Attack Tracker history to verify whether a target already fell.
  • Restore your latest backup and re-run install if UI panels look empty.
